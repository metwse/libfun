libfun

libfun is a personal sandbox library for C utilities. It is not currently production-ready, and probably never will; it simply houses various implementations I find useful.

The library provides two integration strategies:

  • Using Make: As mentioned in the README (see libfun).
  • Header-only library: Using libfun.h.

libfun.h is generated via make headeronly and acts as an stb-style library. This means it is a single file that contains both declarations and implementations. By default, it acts only as a header file. To compile the implementations, you must define the LF_IMPLEMENTATION macro before including it. This gives you full control over your compilation process.

The header-only library is auto-generated by a GitHub workflow and pushed to the headeronly branch. You should grab your copy of libfun.h from there.

Public API

The library provides the following modules:

  • hashmap.h: A hashmap implementation using open addressing and the FNV hash function.
  • map.h: An ordered map implementation using augmented Red-Black trees.
  • stack.h: A standard LIFO stack.

Usage

When using the Make build system, the library prefix is defined during compilation (see libfun). However, when using the header-only version, you must define the prefix before including the header:

#define LF_IMPLEMENTATION
#define LIBFUN_PREFIX lf_
#include "libfun.h"

The default prefix is f. If you do not define a custom prefix, you can use the library like this:

int main(void)
{
struct fhashmap hashmap;
fhashmap_xinit(&hashmap);
// ...
}

Additionally, the header-only library provides more configuration options than the Make-based build system. You can customize the internal behavior of the library by defining specific macros before including libfun.h:

#define LF_IMPLEMENTATION
#define LF_STACK_INITIAL_CAP 128
#include "libfun.h"

For a complete list of macros that can be used to tweak the implementation, please refer to config.h.
